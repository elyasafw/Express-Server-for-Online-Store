# **שרת Express לחנות ספורט (Sport Store Backend)**

פרויקט זה מציג שרת Backend מבוסס Node.js ו-Express המדמה מערכת ניהול עבור חנות ספורט מקוונת. השרת מנהל מוצרים (סינון וחיפוש), עגלות קניות ייעודיות ללקוחות, ביצוע הזמנות (Checkout) ועדכון מלאי ויתרות כספיות בזמן אמת.

## **🧭 נתיבי ה-API (Endpoints)**

💡 **הערת הגשה לגבי מבנה הנתיבים:**

לצורך ארגון קוד מודולרי ונקי, בוצעה חלוקה של השרת לראוטרים נפרדים (customerRouter, orderRouter, productRouter). עקב כך, נתיבי המקור שונו וסודרו תחת תחיליות לוגיות ברורות (/customers, /orders, /products). שינוי זה מפריד בצורה מקצועית בין היישויות השונות בשרת.

### **ראוטר מוצרים (/products)**

| **מתודה** | **נתיב** |                  **תיאור**                  |                             **פרמטרים (Query Params)**                             |
| :------------------: | :----------------: | :----------------------------------------------------: | :---------------------------------------------------------------------------------------------: |
|    **GET**    |     /products     | קבלת רשימת מוצרי ספורט מסוננת | inStock (true/false), maxPrice (מחיר מקסימלי), search (חיפוש בשם המוצר) |

### **ראוטר לקוחות ועגלה (/customers)**

| **מתודה** |        **נתיב**        |                     **תיאור**                     | **פרמטרים / גוף הבקשה (Body)** |
| :------------------: | :------------------------------: | :-----------------------------------------------------------: | :--------------------------------------------------: |
|    **GET**    |         /customers/cart         |        הצגת עגלת הקניות של הלקוח        |                 ?customerId=CUST-101                 |
|    **GET**    |    /customers/account/balance    | הצגת יתרת החשבון הנוכחית של הלקוח |                 ?customerId=CUST-101                 |
|    **POST**    |      /customers/cart/items      |    הוספת מוצר ספורט לעגלת הקניות    |  JSON Body (מזהה לקוח, מוצר וכמות)  |
|   **DELETE**   | /customers/cart/items/:productId |    הסרת מוצר ספציפי מעגלת הקניות    | Path Param productId\+ JSON Body (מזהה לקוח) |

### **ראוטר הזמנות (/orders)**

| **מתודה** | **נתיב** |                    **תיאור**                    | **פרמטרים / גוף הבקשה (Body)** |
| :------------------: | :----------------: | :--------------------------------------------------------: | :-------------------------------------------------: |
|    **GET**    |      /orders      | הצגת היסטוריית ההזמנות של הלקוח |                ?customerId=CUST-101                |
|    **POST**    |  /orders/checkout  |  ביצוע רכישה סופית ויצירת הזמנה  |            JSON Body (מזהה לקוח)            |

## **💼 חוקים עסקיים מרכזיים**

1. **נגישות מוצרים:** קבלת רשימת מוצרים וחיפושם (GET /products) פתוחה לכלל המשתמשים ואינה דורשת מזהה לקוח (customerId).
2. **זיהוי לקוח ללא מערכת התחברות:** כל הבקשות הנוגעות לעגלה, יתרה והזמנות דורשות מזהה לקוח מפורש (customerId) כחלק מה-Query Params או ה-Body.
3. **ניהול מלאי בעת הוספה לעגלה:** ניתן להוסיף מוצר לעגלה רק אם הוא קיים במלאי. המלאי הפיזי בקובץ הנתונים יורד **אך ורק** לאחר ביצוע תהליך Checkout מוצלח.
4. **רישום אוטומטי של לקוח חדש:** במידה ונשלחת בקשת Checkout עבור לקוח שאינו קיים במערכת (מזהה חדש), השרת יוצר עבורו באופן אוטומטי רשומת לקוח חדשה ומעניק לו את יתרת הפתיחה המוגדרת בקובץ ה-.env.
5. **מגבלות תהליך הרכישה (Checkout):** תהליך הרכישה ייכשל ויחזיר שגיאה (קוד 400\) במקרים הבאים:
   * עגלת הקניות של הלקוח ריקה.
   * כמות של אחד המוצרים בעגלה גדולה מהמלאי הקיים בפועל.
   * סך ההזמנה הכולל גבוה מיתרת החשבון הנוכחית של הלקוח (balance).

## **🚀 הוראות התקנה והרצה**

### **1\. הגדרת משתני סביבה (Environment Variables)**

יש ליצור קובץ בשם .env בתיקיית השורש של הפרויקט (באותו הגובה של app.js) ולהגדיר את המשתנים הבאים:

```text
PORT=   מספר הפורט של השרת
DB\_BASE\_PATH=   נתיב המקור של קבצי הדאטה
INITIAL\_BALANCE=   יתרת לקוח התחלתית
```

### **2\. התקנת תלויות**

התקן את חבילות ה-npm הנדרשות לפרויקט על ידי הרצת הפקודה הבאה במסוף:

```bash
npm install
```

### **3\. הרצת השרת**

להפעלת שרת הפיתוח:

```bash
node app.js
```

לאחר ההפעלה, השרת יהיה זמין בכתובת המקומית המוגדרת (לדוגמה: **http://localhost:5000)**.
